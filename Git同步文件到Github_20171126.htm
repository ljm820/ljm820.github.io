Git同步文件到Github

 

一把辛酸泪，终于成功上传

不多说了。。。

 

上传方式：ssh（对于http方式，没试过，建议自行查找资料）

 

环境：

Windows10系统

Git软件

Github网站

 

配置过程：

1、创建Github用户及Repositories，重点是获得相应Repository的ssh地址

1.1 Github用户创建过程不多说，自行注册。

1.2 Repositories创建过程：

首先登录个人Github账号，再进入主页（点击左上角的“猫背影”），找到下图右下角的绿色控件New repository，详见下图

Step 1
 

注：假定此处创建的Repository文件名为gitTestFile

 

2、git软件上创建密钥并将公钥与Github关联上

2.1登录git bash，用命令行进行操作

命令为：$ssh-keygen -t rsa -C “Github创建时所用邮箱”

2.2找到本地密钥，并复制is_rsa.pub中文本到Github网页的SSH keys内（注：SSH keys地址： Settings->Personal settings->SSH and GPG keys）

测试显示关联成功

 

3、将创建的Repository同步到本地后再于该文件夹中创建文件并推送上去

#注：以下命令均在git bash中输入执行

3.1创建并进入本地Repository文件

$cd /d/

$mkdir gitRepo

$cd gitRepo

3.2同步Github中的Repository到本地

$git remote add origin git@github.com:username/gitTestFile.git        # username为自己的用户名，gitTestFile为假定的Repository文件名

$git clone -o gitTestFile git@github.com: username/gitTestFile.git

 

3.3创建一个测试文件并于本地Repository上推送

$cd gitTestFIle                                         #★★★进入了本地的Repository文件★★★

$echo “#test file” >> README.md            #创建文件README.md

$git init

$git add README.md                              #添加README.md进入本地缓存库

$git commit -m “first commit”                  #给README.md文件添加注释

$git push -u origin master                       #★★★推送README.md文件到Github★★★

最后与Github网页上按F5刷新即可查看最新信息
