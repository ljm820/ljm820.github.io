基于 Starship 的 PowerShell 7 终端美化配置实践与工具链集成方案

一、为什么又要折腾终端（Hypothesis）

用 Windows 做开发这几年，PowerShell 默认那副"白底黑字 + 蓝色提示符"的样子一直没怎么变。平时敲 git status、ssh 跳板机、跑 Python 脚本，倒也能用，但只要切到 WSL 或者 macOS 上看一眼别人那花里胡哨的提示符，再回来就总觉得哪里别扭。

真正让我下定决心动手的，是两件小事：

第一件是某次在仓库里切分支，连续敲了三次 git branch 才确认当前在哪个分支上——因为提示符里压根不显示。第二件是同事发来一段带 emoji 和图标的终端截图，我这边却显示成一堆方框。查了才知道是字体没装 Nerd Font。

所以这次配置的目标其实很朴素：

  • 提示符里能直接看到当前 Git 分支、Python 虚拟环境、命令执行耗时；
  • 不再出现图标方框、中文乱码；
  • 补全体验接近 fish / zsh + zsh-autosuggestions 的水准；
  • 顺手把 ls、cat、find 这些老命令换成现代替代品。

至于为什么最后选了 Starship 而不是更主流的 Oh My Posh，下面单独说。

二、Starship 与 Oh My Posh：选型对比（Analysis）

这两个工具几乎是 Windows 终端美化绕不开的两个名字，社区里也经常有人问"到底选哪个"。我自己两个都装了，先用 Oh My Posh 跑了一周，又切到 Starship 跑了两周，下面是对比结论。

2.1 两者定位与技术栈

维度 Oh My Posh Starship
实现语言 Go Rust
配置格式 JSON / YAML / TOML（推荐 JSON） TOML
跨 Shell 支持 PowerShell、CMD、Bash、Zsh、Fish、Nu 等 几乎所有主流 Shell
主题生态 内置 150+ 主题，开箱即用 默认主题较少，强调自定义
启动开销 略高（实测冷启动多 30~80ms） 较低（Rust + 异步渲染）
配置复杂度 主题切换一行命令，深度定制需写 JSON 一份 starship.toml 搞定
字体依赖 强依赖 Nerd Font 强依赖 Nerd Font

2.2 实际使用感受

Oh My Posh 的优势在于主题多。oh-my-posh get themes 一敲，一百多个主题随便挑，agnoster、powerlevel10k_rainbow 这些经典样式都有现成的。对于不想折腾配置、只想快速好看的人来说，它确实更友好。

但我在用 Oh My Posh 的过程中遇到两个比较烦的问题：

一是 JSON 配置写起来太啰嗦。想改个段（segment）的顺序、加个条件显示，得嵌套好几层 objects 和 arrays，稍微复杂一点就容易写错，而且报错信息不太友好。

二是冷启动有可感知的延迟。我的 $PROFILE 里挂了 PSReadLine、Terminal-Icons、PSCompletions 几个模块，再加上 Oh My Posh，新开一个 Windows Terminal 标签页时，提示符出现前能明显感觉到"顿"一下。换成 Starship 之后这个顿挫感消失了。

Starship 的 TOML 配置可读性好很多，比如下面这段就是我目前在用的：

# ~/.config/starship.toml
add_newline = true
format = """$directory$git_branch$git_status$python$character"""

[character]
success_symbol = "[➜](bold green)"
error_symbol = "[✗](bold red)"

[git_branch]
symbol = " "

[python]
symbol = " "
format = '[${symbol}${pyenv_prefix}(${version})( \($virtualenv\))]($style) '

一目了然，改起来也快。

2.3 选型结论

如果你追求开箱即用的炫酷主题，且不在意几十毫秒的启动延迟，选 Oh My Posh。

如果你更看重启动速度、配置可读性、跨平台一致性（比如同时用 Windows + WSL + macOS），选 Starship。

我最终留在 Starship，主要原因是后者。但 Oh My Posh 我没卸载，oh-my-posh 命令本身在字体安装（oh-my-posh font install）这件事上还是最方便的，所以保留作为辅助工具。

三、安装与基础配置（Method）

下面进入实操环节。整体顺序是：装 PowerShell 7 → 装字体 → 装美化引擎 → 装插件 → 写 $PROFILE。每一步都踩过坑，对应说明都写在后面。

3.1 安装 PowerShell 7

Windows 10/11 自带的 Windows PowerShell 5.1 基于 .NET Framework，已经停止功能更新。PowerShell 7 基于 .NET Core，跨平台、性能更好，且 $PROFILE 路径与 5.1 完全独立，互不干扰。

winget install Microsoft.PowerShell

安装完成后，Windows Terminal 会自动识别并新增 "PowerShell 7" 配置项（图标是黑色的，区别于蓝色的 Windows PowerShell）。建议在 Windows Terminal 设置里把它设为默认配置文件。

坑点：PowerShell 5.1 与 7 的 $PROFILE 路径不同。 - 5.1：C:\Users\$YourName\Documents\WindowsPowerShell\Microsoft.PowerShell_profile.ps1 - 7.x：C:\Users\$YourName\Documents\PowerShell\Microsoft.PowerShell_profile.ps1 注意中间目录名一个是 WindowsPowerShell，一个是 PowerShell，网上很多老教程抄的是 5.1 的路径，在 7 里改了不生效。

3.2 安装 Nerd Font（解决图标方框、乱码）

这是整个美化流程里最容易被忽略、但最关键的一步。Starship、Oh My Posh、Terminal-Icons 这些工具都依赖 Nerd Font 提供的图标字形（Powerline 箭头、Git 图标、语言图标等）。不装字体，提示符里就是一堆 □。

最省事的方式是用 Oh My Posh 自带的字体安装命令：

oh-my-posh font install Meslo

会弹出交互菜单，选择 Meslo 系列即可（也可以选 FiraCode Nerd Font、JetBrainsMono Nerd Font，看个人喜好）。

如果 Oh My Posh 没装，也可以手动去 Nerd Fonts Releases 下载 zip，解压后右键"为所有用户安装"。

3.3 在终端与编辑器中应用字体

字体装好了还不够，得让终端真正用上它。

Windows Terminal：

设置 → 配置文件 → 默认值 → 外观 → 字体 → 选择 MesloLGM Nerd Font（或你刚装的那个）。

Cursor / VS Code 内置终端：

设置 → 搜索 Terminal › Integrated: Font Family → 填入 MesloLGM Nerd Font。

这里有个小技巧：在 VS Code 的 settings.json 里可以直接写： `json "terminal.integrated.fontFamily": "MesloLGM Nerd Font", "terminal.integrated.fontSize": 13 ` 比在 GUI 里点快得多。

3.4 安装美化引擎与插件

# Oh My Posh（保留作为字体管理工具，未启用为提示符引擎）
winget install JanDeDobbeleer.OhMyPosh

# Starship（实际启用的提示符引擎）
winget install Starship.Starship

插件方面，推荐安装以下几个：

模块 作用 安装命令
PSReadLine 已内置，但需升级到 2.3+ 才支持预测补全 Install-Module PSReadLine -Force
CompletionPredictor 提供 Plugin 来源的预测建议 Install-Module CompletionPredictor -Force
PSCompletions 多语言智能补全（Git、Docker、Python 等） Install-Module PSCompletions -Force
Terminal-Icons 为 ls / Get-ChildItem 输出加文件类型图标 Install-Module Terminal-Icons -Force

3.5 配置 `$PROFILE`

先确认配置文件存在，不存在就创建：

if (-not (Test-Path $PROFILE)) { New-Item -Path $PROFILE -Type File -Force }

然后用记事本（或你顺手的编辑器）打开：

notepad $PROFILE

下面是我目前在用的完整配置，注释写得比较细，可以直接抄：

# ============================================================
# PowerShell 7 Profile
# ============================================================

# ---------- 1. 启动 Starship ----------
Invoke-Expression (&starship init powershell)

# ---------- 2. UTF-8 编码（解决中文乱码） ----------
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
$OutputEncoding = [System.Text.Encoding]::UTF8
chcp 65001 | Out-Null

# ---------- 3. PSReadLine：补全与历史 ----------
Import-Module PSReadLine

Set-PSReadlineKeyHandler -Key Tab -Function Complete
Set-PSReadlineKeyHandler -Key "Ctrl+d" -Function MenuComplete
Set-PSReadlineKeyHandler -Key "Ctrl+z" -Function Undo
Set-PSReadlineKeyHandler -Key UpArrow -Function HistorySearchBackward
Set-PSReadlineKeyHandler -Key DownArrow -Function HistorySearchForward

# 预测建议来源：History + Plugin 共同提供
# 可选值：None / History / Plugin / HistoryAndPlugin
Set-PSReadLineOption -PredictionSource HistoryAndPlugin

# 以清单面板方式在光标下方显示多个建议 (ListView)
# 若偏好行内灰色字体，可改用：InlineView
Set-PSReadLineOption -PredictionViewStyle ListView

# ---------- 4. 文件图标 ----------
Import-Module Terminal-Icons

# ---------- 5. sudo 别名（依赖 gsudo） ----------
Set-Alias sudo gsudo

# ---------- 6. which 函数 ----------
Function WHICHCMD {
    $(Get-Command $args[0]).Source;
}
Set-Alias -Name which -Value WHICHCMD

保存后重开终端，提示符应该就变成 Starship 的样式了。如果还是方框，回到 3.2 检查字体。

四、进阶：现代 CLI 工具链集成（Method 续）

基础美化搞定后，下一步是把那些"上古命令"换成现代替代品。这一步收益其实比美化提示符还大——日常敲命令的体感提升主要来自这里。

4.1 现代 CLI 工具推荐

传统命令 现代替代 优势
ls eza 彩色输出、Git 状态集成、树形视图
cat bat 语法高亮、行号、分页
find fd 速度快、语法简洁、默认忽略 .gitignore
grep ripgrep (rg) 极速、默认递归、默认忽略二进制
- fzf 通用模糊查找器，配合任何列表都能用

安装命令：

winget install eza-community.eza
winget install sharkdp.bat
winget install sharkdp.fd
winget install BurntSushi.ripgrep.MSVC
winget install fzf

装完之后，在 $PROFILE 里加几行别名，让新工具无缝替代老命令：

# ---------- 7. 现代 CLI 别名 ----------
Set-Alias -Name ls -Value eza
Set-Alias -Name cat -Value bat
Set-Alias -Name find -Value fd

注意：Set-Alias ls eza 会覆盖内置 ls，如果偶尔需要原版行为，可以用 Get-ChildItem 或 dir。

4.2 VS Code / Cursor 终端配置

VS Code 系编辑器的内置终端会继承 Windows Terminal 的部分行为，但有几个独立设置需要单独配。打开 settings.json（Ctrl+Shift+P → "Preferences: Open User Settings (JSON)"），加入：

{
    "terminal.integrated.defaultProfile.windows": "PowerShell",
    "terminal.integrated.fontFamily": "MesloLGM Nerd Font",
    "terminal.integrated.fontSize": 13,
    "terminal.integrated.scrollback": 10000,
    "terminal.integrated.copyOnSelection": true,
    "terminal.integrated.rightClickBehavior": "paste"
}

其中 copyOnSelection 和 rightClickBehavior 两个选项对习惯 Linux 终端操作的人特别友好——选中即复制、右键即粘贴，省去 Ctrl+C / Ctrl+V 的来回切换。

4.3 pyenv-win：多版本 Python 管理

做 Python 开发免不了要切版本。Windows 上对应 pyenv 的实现是 pyenv-win，安装方式：

# 方式一：通过 PowerShell 脚本
Invoke-WebRequest -UseBasicParsing -Uri "https://raw.githubusercontent.com/pyenv-win/pyenv-win/master/pyenv-win/install-pyenv-win.ps1" -OutFile "./install-pyenv-win.ps1"; &"./install-pyenv-win.ps1"

安装完成后，Starship 的 [python] 段会自动识别 pyenv 当前版本，并在提示符里显示。常用命令：

pyenv install --list       # 列出可安装版本
pyenv install 3.12.4       # 安装指定版本
pyenv global 3.12.4        # 设置全局默认
pyenv local 3.11.9         # 为当前目录设置版本

坑点：pyenv-win 安装后需要重启终端，且要确认环境变量 PYENV 和 PYENV_HOME 已正确写入。如果 pyenv 命令找不到，多半是环境变量没生效，重启 Windows Terminal 试试。

4.4 gsudo：Windows 上的 sudo

Linux/macOS 用惯了 sudo，回 Windows 还要右键"以管理员身份运行"实在别扭。gsudo 解决的就是这个问题：

winget install gerardog.gsudo

装完之后，$PROFILE 里那句 Set-Alias sudo gsudo 就生效了。直接 sudo winget upgrade --all 即可以管理员权限执行命令，体验和 Linux 几乎一致。

4.5 在 Windows Terminal 中新增"PowerShell 7（管理员）"选项

虽然有了 gsudo，但有时候还是想直接开一个管理员终端。Windows Terminal 支持自定义配置文件，操作步骤：

  • 打开 Windows Terminal → 设置（Ctrl+,）→ 左下角"添加新配置文件"。
  • "新建空配置文件"，名称填 PowerShell 7 (Admin)。
  • 命令行填：gsudo.exe pwsh.exe -NoLogo（如果没有 gsudo，可以填 pwsh.exe -NoLogo，但需要勾选"以管理员身份运行此配置文件"）。
  • 图标可以自定义，我用了 ms-appx:///ProfileIcons/pwsh.png。
  • 保存后，下拉菜单里就多了一个管理员选项。

五、踩坑记录与最终效果（Findings）

整套配置跑下来，有几个坑值得单独记一下，免得下次重装系统再踩。

5.1 常见问题排查

问题一：提示符里全是方框 □。

原因：终端字体不是 Nerd Font。回到 3.2 装字体，3.3 在终端里选字体。注意 VS Code 内置终端要单独设。

问题二：中文输出乱码。

原因：控制台编码不是 UTF-8。$PROFILE 里那三行 [Console]::OutputEncoding / $OutputEncoding / chcp 65001 是关键，缺一不可。只设 chcp 65001 在 PowerShell 7 里有时不够。

问题三：$PROFILE 改了不生效。

原因：改错文件了。PowerShell 7 的路径是 Documents\PowerShell\，不是 Documents\WindowsPowerShell\。用 echo $PROFILE 确认实际路径。

问题四：PSReadLine 预测补全不显示。

原因：PSReadLine 版本太低。Get-Module PSReadLine 看版本，低于 2.3.0 的话 Install-Module PSReadLine -Force 升级一下。PredictionSource 和 PredictionViewStyle 这两个选项是 2.3.0 才加的。

问题五：Starship 启动后提示符闪烁。

原因：starship.toml 里 add_newline = true 配合某些主题会有重绘。把 add_newline 改成 false，或者升级 Starship 到最新版（winget upgrade Starship.Starship）。

5.2 最终效果

配置完成后的终端大致长这样：

  • 第一行：当前目录路径（带文件夹图标）+ Git 分支（带分支图标）+ Python 版本（带蛇图标）
  • 第二行：➜ 提示符，命令成功为绿色，失败为红色
  • 输入命令时：下方浮出 ListView 风格的预测建议，按 → 接受
  • ls 输出：彩色文件名 + 文件类型图标（文件夹、Python 文件、JSON 文件各有不同图标）
  • cat 输出：语法高亮 + 行号

启动耗时方面，用 Measure-Command { pwsh -NoLogo -Command "exit" } 测了一下，冷启动约 280ms，比之前挂 Oh My Posh 时快了大概 60ms。日常使用感知不到延迟。

5.3 一点反思

折腾完这一套，最大的感受是：美化是手段，不是目的。

最初确实是为了好看才动手，但真正提升效率的，其实是 PSReadLine 的预测补全、eza 的 Git 状态集成、ripgrep 的搜索速度这些东西。提示符好看只是顺带的副产品。

另外，配置文件一定要版本管理。我的 $PROFILE 和 starship.toml 都丢进了 dotfiles 仓库，换机器时 git clone + 软链接一下就能恢复全部环境，比手动重配省心太多。
